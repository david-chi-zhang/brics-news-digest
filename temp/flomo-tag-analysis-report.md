## ⚠️ 发现的问题

### 1. 标签体系过于细化
- **经济** 大类下有 15 个子标签，但多数使用频率极低
- 例如：`经济/汇率`、`经济/黄金`、`经济/国际投资`、`经济/产业政策` 等都只有 1 次使用
- 这导致标签体系臃肿，检索效率低

### 2. 标签层级不一致
- 大部分标签采用 `分类/子分类` 结构（如 `经济/能源`）
- 但部分标签是扁平的（如 `指标`、`思考`、`气候`、`国际政治`）
- `南非` 作为独立标签，与 `区域/非洲` 层级不一致

### 3. 标签使用率偏低
- 56 个注册标签中，只有 28 个被使用（50%）
- 大量标签创建后从未使用，如：
  - `区域/东南亚`、`区域/中东`、`区域/南亚/孟加拉`
  - `经济/半导体`、`经济/科技`、`经济/财政`、`经济/金融`
  - `读书/Principles_Debt_Crisis`、`读书/Railpolitik` 等 5 个读书子标签
  - `人工智能`、`美食`、`金句`、`自我吐槽`

### 4. 2 条笔记无标签
- 约 4% 的笔记没有打标签，影响后续检索和知识关联

### 5. 读书标签过于具体
- `读书/浪潮将至` 使用了 4 次，但其他读书子标签都是 0 次
- 建议：读书类标签可以简化为 `读书/书名`，但需要评估是否有长期价值

### 建议 4: 读书标签策略优化

**当前问题**: 6 个读书子标签中，5 个从未使用

**建议方案**:

**方案 A - 简化**: 只保留 `读书` 一级标签，书名写在笔记正文中
```
#读书
《浪潮将至》读书笔记...
```

**方案 B - 按需创建**: 不预先创建读书子标签，读到第 2 本书时再创建 `读书/书名`

**推荐**: 方案 A，因为：
- Flomo 支持全文搜索，搜索书名即可找到
- 避免标签体系膨胀
- 一本书的笔记通常较少，单独标签价值有限

### 建议 6: 建立标签使用规范

**标签使用原则**:

1. **3 次原则**: 一个标签使用满 3 次后才保留，否则合并到父级
2. **MECE 原则**: 标签之间相互独立、完全穷尽，避免重叠
3. **检索导向**: 标签是为了检索，不是为了分类美观
4. **适度层级**: 最多 2 级层级（如 `经济/能源`），避免 3 级+（如 `经济/大宗商品/铜`）

**新标签创建流程**:
```
1. 先搜索现有标签是否可用
2. 优先使用现有父级标签（如 经济/能源）
3. 只有当某个子标签预计会使用 3 次以上时，才创建新子标签
4. 定期（每季度）审查标签使用情况，合并低频标签
```
